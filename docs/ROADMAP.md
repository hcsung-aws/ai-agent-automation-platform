# 로드맵 및 개선 계획

## 현재 버전 (v1.0) 한계점

### 1. 점진적 개선 사이클 미완성

| 목표 | 현재 상태 | 부족한 점 |
|------|----------|----------|
| 실행 기록 저장 | ✅ 완료 | - |
| 실행 기록 검토 | ⚠️ 기본 UI | 피드백 수집 기능 없음 |
| KB/지침 반영 | ⚠️ 수동 | 자동 제안 기능 없음 |
| 개선된 Agent | ❌ 미구현 | 피드백 기반 학습 없음 |

### 2. 운영 편의성 부족

- 로그 검토 UI가 단순 (테이블 뷰만 제공)
- 피드백 수집 메커니즘 없음
- KB 문서 작성 가이드 부족

### 3. 프로덕션 준비 미흡

- 인증/권한 관리 없음
- 멀티 테넌트 미지원
- AgentCore 런타임 배포 미완료

## 개선 로드맵

### v1.1 - 검토 기능 강화 (2주)

**목표**: 실행 기록 검토 → 피드백 수집 → KB 개선 제안

#### 작업 항목

1. **실행 기록 검토 UI 개선**
   - 세션별 대화 흐름 시각화
   - 도구 호출 상세 보기
   - 필터링 (날짜, Agent 타입, 상태)

2. **피드백 수집 기능**
   - 응답별 좋아요/싫어요 버튼
   - 코멘트 입력 기능
   - 피드백 저장 (DynamoDB)

3. **KB 문서 제안**
   - 자주 실패하는 질문 패턴 분석
   - KB 문서 템플릿 자동 생성
   - 관리자 검토 후 KB 반영

#### 예상 결과물
```
logs_api.py 개선:
- GET /sessions/{id}/feedback (피드백 조회)
- POST /sessions/{id}/feedback (피드백 저장)
- GET /suggestions (KB 문서 제안 목록)
```

---

### v1.2 - 자동 개선 (2주)

**목표**: 피드백 기반 자동 개선 제안

#### 작업 항목

1. **실행 패턴 분석**
   - 성공/실패 패턴 클러스터링
   - 자주 사용되는 도구 조합 분석
   - 응답 시간 이상치 탐지

2. **자동 개선 제안**
   - 실패 케이스 → KB 문서 초안 생성
   - 반복 질문 → FAQ 문서 제안
   - System Prompt 개선 제안

3. **A/B 테스트 프레임워크**
   - System Prompt 버전 관리
   - 성능 비교 대시보드

#### 예상 결과물
```
src/utils/pattern_analyzer.py (신규)
src/utils/improvement_suggester.py (신규)
```

---

### v2.0 - 프로덕션 (4주)

**목표**: 실제 운영 환경 배포

#### 작업 항목

1. **인증/권한 관리**
   - Cognito 연동
   - 역할 기반 접근 제어 (RBAC)
   - API Key 관리

2. **멀티 테넌트**
   - 조직별 데이터 격리
   - 조직별 KB 관리
   - 사용량 추적

3. **AgentCore 런타임 배포**
   - 컨테이너 이미지 빌드
   - AgentCore 배포 자동화
   - 오토스케일링 설정

4. **A2A 프로토콜 연동**
   - 외부 Agent와 통신
   - Agent 디스커버리
   - 프로토콜 어댑터
